# 수동 QA 체크리스트 — 상주 FTXUI TUI (v2)

TUI(`leaves/ui` = `planning::ui::TuiApp`)는 GUI 환경 경계라 단위 테스트 대상에서 제외한다(설계 02 TDD 계획 C). 대신 사람이 직접 띄워 확인하는 **인수 테스트(UAT) 체크리스트**다. 각 항목은 자동 단위 테스트로 가드되지 않는 TUI 거동만 담는다(유스케이스/도메인/toolshed는 단위 테스트가 가드).

## 사용법

1. 빌드 후 무인자로 실행해 상주 TUI를 띄운다 (서브커맨드 없이 실행 = 대시보드 진입).
2. 아래 항목을 위에서부터 하나씩 재현하고, 기대 결과가 맞으면 `[x]`로 체크한다.
3. 실패하면 항목 옆에 증상을 적고 이슈로 옮긴다.
4. 데이터 흐름 근거는 handoff.md 5장(5-2 충돌 / 5-3 검증 / 5-4 에러 / 5-5 포커스)을 참조.

---

## 1. 키 컨텍스트 / 액션바 (5-5)

- [ ] **패널 전환 시 액션바 가변** — 1-4 또는 화살표로 활성 패널을 바꾸면 하단 액션바가 해당 패널의 가능한 동작으로 바뀐다.
- [ ] **같은 키, 다른 의미** — 같은 키(예: `d`, `l`)가 활성 패널에 따라 다른 동작을 한다(Todos의 `d`=완료, Goals의 `l`=+1 기록 등).
- [ ] **Today 읽기전용** — Today 패널에서는 쓰기 키(a/e/d/x/l)가 무반응이다(중복 조작 방지).

## 2. 충돌 흐름 2-phase (5-2, event add/edit 한정)

- [ ] **겹침 추가 시 충돌 모달** — 기존 이벤트와 시간이 겹치는 이벤트를 추가하면 충돌 모달이 뜨고, 아직 저장되지 않은 상태다.
- [ ] **강행 시 저장** — 충돌 모달에서 추가를 강행하면 저장되고 영향 패널이 갱신된다.
- [ ] **취소 시 미저장** — 충돌 모달에서 취소하면 모달이 닫히고 저장되지 않는다.
- [ ] **경계 맞닿음은 충돌 아님** — 앞 이벤트 끝 시각 = 뒤 이벤트 시작 시각(back-to-back)이면 충돌 모달 없이 바로 저장된다.
- [ ] **수정 시 재검사** — 이벤트 시간을 겹치도록 수정하면 동일한 충돌 모달이 뜬다(시간 변경 시에만).

## 3. 입력 검증 2층 (5-3)

- [ ] **구문 오류는 모달 유지** — 파싱 불가한 입력(잘못된 날짜 형식 등)을 넣으면 해당 필드에 인라인 에러가 뜨고 모달이 유지된다(유스케이스 호출 안 됨).
- [ ] **불변식 위반은 배너** — 파싱은 되나 도메인 규칙 위반(start>end, 빈 제목, target<=0 등)이면 모달 배너로 에러가 뜨고 모달이 유지된다.
- [ ] **성공만 모달 닫힘** — 정상 입력일 때만 모달이 닫히고 패널이 갱신된다.

## 4. 에러 모델 — 루프 생존 (5-4)

- [ ] **액션 에러로 안 죽음** — 액션 처리 중 예외가 나도 토스트/배너로 표시되고 로그에 남으며, TUI 루프는 계속 돈다(종료되지 않음).
- [ ] **부트스트랩 실패는 루프 전 종료** — DB/설정/마이그레이션 실패 시 stderr에 메시지를 내고 루프 진입 전에 종료한다(TUI가 뜨지 않음).

## 5. 빈 상태 / 스크롤 (5-5)

- [ ] **빈 패널은 추가만** — 항목이 없는 패널에서는 `a`(추가)만 활성이고 placeholder가 보인다.
- [ ] **패널 내 스크롤** — 항목이 화면보다 많으면 명시적 페이지 없이 패널 안에서 스크롤되며, 패널마다 스크롤이 독립적이다.

## 6. 종료

- [ ] **q 정상 종료** — `q`로 루프가 종료되고 teardown 후 종료코드 0으로 빠진다.

---

> 갱신 규칙: TUI 거동/흐름이 바뀌면 이 체크리스트도 같이 갱신한다. 항목이 자동 단위 테스트로 흡수되면(예: 표현 로직 seam을 나중에 추출) 그 항목은 여기서 제거하고 TDD 계획으로 옮긴다.
